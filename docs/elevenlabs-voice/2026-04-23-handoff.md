## What was decided

Built a complete spec and plan for a personal TTS system that gives Claude a voice via ElevenLabs. Two automatic integration paths:

- **Claude Code CLI** — stop hook fires after every response, filters prose, speaks it
- **Claude Desktop** — UIAutomation daemon watches accessibility tree, speaks sentence-by-sentence as Claude streams

Voice selection persists between sessions in `~/.claude/voice-config.json`. Slash commands (`/voice on`, `/voice off`, `/voice list`, `/voice pick`) control everything from inside Claude Code.

## Verbosity feature (added at end of session)

Two-layer approach — no writes to CLAUDE.md, takes effect immediately in the current conversation:

**Layer 1 (primary): In-conversation directive**  
`/voice verbosity low|medium|high` runs the config script, then the slash command markdown file instructs Claude to adopt the verbosity rule immediately in the current session. No new session required. No CLAUDE.md writes ever.
- `low` — Claude summarizes lists of 3+ items in prose, max 2 bullets per response
- `medium` — Claude limits lists to 5 items, summarizes the rest
- `high` — no constraints

**Layer 2 (fallback): `filter.py` post-processing**  
Still truncates lists in the TTS text. Catches cases where Claude doesn't follow the directive.

`voice_verbosity.py` only writes to `config.json`. Users who want persistence across sessions can add the instruction to their own CLAUDE.md manually. Tests in Task 2 (config) and Task 3 (filter).

**Claude Desktop verbosity — Task 12 (MCP prompt server)**  
A local MCP server (`mcp_server/voice-mcp-server.py`) exposes three named prompts in Claude Desktop's prompt picker UI. Selecting one injects the verbosity directive into the conversation and updates `config.json`. Same effect as `/voice verbosity` in Claude Code. `setup.py` registers the server in `%APPDATA%\Claude\claude_desktop_config.json` automatically. Requires one Claude Desktop restart after setup.

## Key design decisions made

- **No system prompts** for Claude Desktop — UIAutomation watches the window directly instead
- **Sentence-level streaming** for Desktop (speaks as Claude types), **full-response** for Claude Code (stop hook fires once)
- **Filter strips**: triple-backtick code fences, inline code, tool call blocks, markdown headers, horizontal rules — only prose spoken
- **Config at** `~/.claude/voice-config.json` — shared by hook and daemon, voice_id persists
- **ffplay** (bundled with ffmpeg) handles MP3 playback
- **Task 8 (tree discovery)** must run before Task 9 (daemon) — needs Claude Desktop open to inspect accessibility tree
