## What was decided

Built a complete spec and plan for a personal TTS system that gives Claude a voice via three providers: ElevenLabs, OpenAI, or Google Cloud TTS. Two automatic integration paths:

- **Claude Code CLI** — stop hook fires after every response, filters prose, speaks it
- **Claude Desktop** — UIAutomation daemon watches accessibility tree, speaks sentence-by-sentence as Claude streams

Voice selection persists between sessions in `~/.claude/voice-config.json`. Slash commands (`/voice on`, `/voice off`, `/voice list`, `/voice pick`, `/voice provider`, `/voice verbosity`) control everything from inside Claude Code.

**This is a public GitHub project.**

## Verbosity (finalized)

Three levels, two-layer enforcement:

**Layer 1 (primary): in-conversation directive** — `/voice verbosity` injects a rule into the current Claude session so Claude generates less verbose output.

**Layer 2 (fallback): filter.py post-processing** — catches cases where Claude ignores the directive.

| Level | filter.py behavior |
|---|---|
| `low` (default) | Replaces entire bullet list with "The response included a list of N items." No bullets spoken. |
| `medium` | Keeps first 3 bullets, then "... and N more." |
| `high` | No truncation — reads everything |

## Voice list behavior

- **ElevenLabs**: `/voice list` → `category=premade` (~30-50 official voices). `/voice list all` → everything on the account including cloned/custom. Community library (10k+) not accessible via API on free tier.
- **Google**: `/voice list` → en-US Neural2 voices only (fetched from `GET /v1/voices?languageCode=en-US`, filtered to Neural2). ~12 voices.
- **OpenAI**: 6 fixed voices — alloy, echo, nova, onyx, shimmer, fissure. No list command needed.

### Task 4: tts.py ⬅️ START HERE
**Files:** `src/tts.py`, `tests/test_tts.py`

Three-provider TTS engine + threaded ffplay audio queue.

**API details:**

**ElevenLabs:**
```
POST https://api.elevenlabs.io/v1/text-to-speech/{voice_id}
Headers: xi-api-key: {api_key}
Body: {"text": "...", "model_id": "eleven_monolingual_v1"}
Response: raw MP3 bytes (stream)
```

**OpenAI:**
```
POST https://api.openai.com/v1/audio/speech
Headers: Authorization: Bearer {api_key}
Body: {"model": "tts-1", "voice": "nova", "input": "..."}
Response: raw MP3 bytes
```

**Google:**
```
POST https://texttospeech.googleapis.com/v1/text:synthesize?key={api_key}
Body: {
  "input": {"text": "..."},
  "voice": {"languageCode": "en-US", "name": "en-US-Neural2-C"},
  "audioConfig": {"audioEncoding": "MP3"}
}
Response: JSON with "audioContent" field (base64-encoded MP3)
```

**Voice list:**
- ElevenLabs: `GET https://api.elevenlabs.io/v1/voices` → `response["voices"]`, filter by `category=="premade"` unless `premade_only=False`
- Google: `GET https://texttospeech.googleapis.com/v1/voices?languageCode=en-US&key={api_key}` → filter where name contains "Neural2"
- OpenAI: hardcoded list `["alloy", "echo", "nova", "onyx", "shimmer", "fissure"]`

**Public interface:**
```python
def speak(text: str, cfg: dict) -> None:
    """Routes to provider based on cfg['provider']. Non-blocking (queues audio)."""

def list_voices(provider: str, api_key: str, premade_only: bool = True) -> list[dict]:
    """Returns list of {id, name} dicts."""
```

**Audio queue:** Background thread consumes MP3 bytes, writes to tempfile, calls `ffplay -nodisp -autoexit -loglevel quiet {path}`. Queue drops oldest item if full (never blocks main thread). `speak()` is non-blocking.

### Task 5: stop-hook.py
`hooks/stop-hook.py` — reads stdin JSON from Claude Code stop event, extracts last assistant message content, calls `filter_response()` + `speak()`.

Stop hook stdin format:
```json
{
  "session_id": "...",
  "transcript": [
    {"role": "user", "content": "..."},
    {"role": "assistant", "content": "the response text"}
  ]
}
```

Extract: last item in `transcript` where `role == "assistant"`, get `content` (may be string or list of `{"type":"text","text":"..."}` blocks — handle both).

## Key design decisions

- **No system prompts** for Claude Desktop — UIAutomation watches the window directly
- **Sentence-level streaming** for Desktop, **full-response** for Claude Code (stop hook fires once)
- **Filter strips**: triple-backtick fences, inline code, tool call blocks, markdown headers, horizontal rules
- **Config** at `~/.claude/voice-config.json` — shared by hook and daemon, atomic writes with filelock
- **ffplay** (bundled with ffmpeg) handles MP3 playback; already installed on user's machine
- **Unclosed fence** strips to EOF — Claude never puts prose after unclosed fence
- **Low verbosity** = zero bullets, just a summary sentence — minimizes ElevenLabs char usage (only 10k/month free)
