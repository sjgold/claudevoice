## What was decided

Built a complete spec and plan for a personal TTS system that gives Claude a voice via three providers: ElevenLabs, OpenAI, or Google Cloud TTS. Two automatic integration paths:

- **Claude Code CLI** — stop hook fires after every response, filters prose, speaks it
- **Claude Desktop** — UIAutomation daemon watches accessibility tree, speaks sentence-by-sentence as Claude streams

Voice selection persists between sessions in `~/.claude/voice-config.json`. Slash commands (`/voice on`, `/voice off`, `/voice list`, `/voice pick`, `/voice provider`, `/voice verbosity`) control everything from inside Claude Code.

**This is a public GitHub project.**

## Verbosity (finalized)

Three levels, two-layer enforcement:

**Layer 1 (primary): in-conversation directive** — `/voice verbosity` injects a rule into the current Claude session so Claude generates less verbose output.

**Layer 2 (fallback): filter.py post-processing** — catches cases where Claude ignores the directive.

| Level | filter.py behavior |
|---|---|
| `low` (default) | Replaces entire bullet list with "The response included a list of N items." No bullets spoken. |
| `medium` | Keeps first 3 bullets, then "... and N more." |
| `high` | No truncation — reads everything |

## Key design decisions

- **No system prompts** for Claude Desktop — UIAutomation watches the window directly
- **Sentence-level streaming** for Desktop, **full-response** for Claude Code (stop hook fires once)
- **Filter strips**: triple-backtick fences, inline code, tool call blocks, markdown headers, horizontal rules
- **Config** at `~/.claude/voice-config.json` — shared by hook and daemon, atomic writes with filelock
- **ffplay** (bundled with ffmpeg) handles MP3 playback; already installed on user's machine
- **Unclosed fence** strips to EOF — Claude never puts prose after unclosed fence
- **Low verbosity** = zero bullets, just a summary sentence — minimizes ElevenLabs char usage (only 10k/month free)
